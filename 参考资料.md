# ParentDocumentRetriever

**`ParentDocumentRetriever` 是一种高级检索器，通过在检索阶段使用“小块”数据确保精度，而在生成阶段返回其对应的“大块”父文档来提供完整的上下文。**

它解决了 RAG 中的一个核心矛盾：为了检索精准，分块通常需要很小；但为了让 LLM 生成高质量答案，分块需要足够大以包含完整的逻辑。

## 核心机制

`ParentDocumentRetriever` 协调以下三个组件：

1.  **Vector Store**: 仅存储“子块”（Child Chunks）的向量，用于精准的语义搜索。
2.  **DocStore**: 存储原始的“父文档”（Parent Documents）或较大的分块。
3.  **Splitters**: 包含子拆分器（用于向量检索）和可选的父拆分器（用于存入 DocStore）。

## 代码示例

**使用 `ParentDocumentRetriever` 时，它不仅负责检索，还内置了文档添加逻辑，会自动处理两层分块及其映射关系。**

```python
from langchain.retrievers import ParentDocumentRetriever
from langchain.storage import InMemoryStore
from langchain_community.vectorstores import Chroma
from langchain_openai import OpenAIEmbeddings
from langchain_text_splitters import RecursiveCharacterTextSplitter

# 1. 定义两层分块策略
# 父分块：最终交给 LLM 的背景信息
parent_splitter = RecursiveCharacterTextSplitter(chunk_size=2000)
# 子分块：用于向量检索，追求极高的匹配精度
child_splitter = RecursiveCharacterTextSplitter(chunk_size=400)

# 2. 初始化存储层
vectorstore = Chroma(collection_name="split_parents", embedding_function=OpenAIEmbeddings())
store = InMemoryStore() # 存储完整的父文档内容

# 3. 创建检索器
retriever = ParentDocumentRetriever(
    vectorstore=vectorstore,
    docstore=store,
    child_splitter=child_splitter,
    parent_splitter=parent_splitter,
)

# 4. 添加文档：它会自动执行两层切分，并将子块向量与父块 ID 关联
retriever.add_documents(docs, ids=None)

# 5. 检索：搜索子块向量，但返回父块 Document 对象
results = retriever.invoke("如何配置 ParentDocumentRetriever?")
```

## 功能总结：添加 vs 检索

`ParentDocumentRetriever` 确实**既可以添加文档，也可以检索文档**。它是对传统检索管道的封装：

*   **添加 (Ingestion)**：调用 `add_documents` 时，它会根据你的 `parent_splitter` 和 `child_splitter` 自动完成切分、嵌入存储以及建立父子映射关系。你不需要手动管理两套库。
*   **检索 (Retrieval)**：调用 `invoke` 时，它会去向量库寻找最相似的“子块”，拿到对应的父 ID，然后从 `docstore` 中取出完整的“大块”返回。

这种“小进大出”的策略是目前 RAG 性能优化的标配方案之一。



## 持久化配置方案

**要使 `ParentDocumentRetriever` 在应用重启后保留已添加的数据，你必须同时持久化其底层依赖的两个组件：向量数据库（Vector Store）和文档存储库（DocStore）。**

默认情况下，如果你使用 `InMemoryStore` 或未指定持久化路径的向量库，数据会在重启后丢失。

你需要将两个组件都指向本地磁盘或持久化服务：

1.  **Vector Store (Chroma)**: 通过 `persist_directory` 将子块向量保存到磁盘。
2.  **DocStore (ByteStore)**: 使用 `LocalFileStore` 将父文档内容保存到本地文件。

```python
from langchain.retrievers import ParentDocumentRetriever
from langchain.storage import LocalFileStore, create_kv_docstore
from langchain_community.vectorstores import Chroma
from langchain_openai import OpenAIEmbeddings
from langchain_text_splitters import RecursiveCharacterTextSplitter

# 1. 设置持久化路径
VEC_DB_PATH = "./chroma_db"
DOC_STORE_PATH = "./parent_docs"

# 2. 初始化持久化向量库
# 重启时，它会自动加载 VEC_DB_PATH 下的向量数据
vectorstore = Chroma(
    collection_name="split_parents",
    embedding_function=OpenAIEmbeddings(),
    persist_directory=VEC_DB_PATH
)

# 3. 初始化持久化文档库
# 使用 LocalFileStore 将原始文档存储在指定目录下
fs = LocalFileStore(DOC_STORE_PATH)
store = create_kv_docstore(fs)

# 4. 初始化检索器
# 由于底层组件已经持久化，重启应用后，检索器会直接关联旧数据
retriever = ParentDocumentRetriever(
    vectorstore=vectorstore,
    docstore=store,
    child_splitter=RecursiveCharacterTextSplitter(chunk_size=400),
    parent_splitter=RecursiveCharacterTextSplitter(chunk_size=2000),
)

# 注意：重启后不需要再次调用 add_documents()，除非有新文档需要加入
# 直接检索即可
results = retriever.invoke("已有的文档内容...")
```

### 关键机制：ID 映射

*   **向量库中的元数据**：当通过检索器添加文档时，每一个子块向量在向量库中都会包含一个指向父文档的 `doc_id`。因为 Chroma 是持久化的，这些元数据映射会被保留。
*   **DocStore 的 Key-Value 存储**：`LocalFileStore` 以 `doc_id` 为文件名，以文档内容为文件内容。检索时，检索器从向量库拿到 `doc_id`，然后去文件系统中寻找对应的文件。

### 生产环境建议

在生产环境中，如果你需要更高的并发能力或分布式支持，建议将底层的存储组件替换为数据库版本：

*   **向量库**：使用部署好的 Chroma 服务、Pinecone 或 Milvus。
*   **文档库**：使用 `RedisStore` 或 `SQLDocStore`。

这样，即使应用多实例重启或迁移，数据依然全局可用。



## add_documents源码

```python
    def _split_docs_for_adding(
        self,
        documents: list[Document],
        ids: list[str] | None = None,
        *,
        add_to_docstore: bool = True,
    ) -> tuple[list[Document], list[tuple[str, Document]]]:
        if self.parent_splitter is not None:
            documents = self.parent_splitter.split_documents(documents)
        if ids is None:
            doc_ids = [str(uuid.uuid4()) for _ in documents]
            if not add_to_docstore:
                msg = "If IDs are not passed in, `add_to_docstore` MUST be True"
                raise ValueError(msg)
        else:
            if len(documents) != len(ids):
                msg = (
                    "Got uneven list of documents and ids. "
                    "If `ids` is provided, should be same length as `documents`."
                )
                raise ValueError(msg)
            doc_ids = ids

        docs = []
        full_docs = []
        for i, doc in enumerate(documents):
            _id = doc_ids[i]
            sub_docs = self.child_splitter.split_documents([doc])
            if self.child_metadata_fields is not None:
                for _doc in sub_docs:
                    _doc.metadata = {
                        k: _doc.metadata[k] for k in self.child_metadata_fields
                    }
            for _doc in sub_docs:
                _doc.metadata[self.id_key] = _id
            docs.extend(sub_docs)
            full_docs.append((_id, doc))

        return docs, full_docs

    def add_documents(
        self,
        documents: list[Document],
        ids: list[str] | None = None,
        add_to_docstore: bool = True,  # noqa: FBT001,FBT002
        **kwargs: Any,
    ) -> None:
        """Adds documents to the docstore and vectorstores.

        Args:
            documents: List of documents to add
            ids: Optional list of IDs for documents. If provided should be the same
                length as the list of documents. Can be provided if parent documents
                are already in the document store and you don't want to re-add
                to the docstore. If not provided, random UUIDs will be used as
                IDs.
            add_to_docstore: Boolean of whether to add documents to docstore.
                This can be false if and only if `ids` are provided. You may want
                to set this to False if the documents are already in the docstore
                and you don't want to re-add them.
            **kwargs: additional keyword arguments passed to the `VectorStore`.
        """
        docs, full_docs = self._split_docs_for_adding(
            documents,
            ids,
            add_to_docstore=add_to_docstore,
        )
        self.vectorstore.add_documents(docs, **kwargs)
        if add_to_docstore:
            self.docstore.mset(full_docs)
```





# 自定义TextSplitter子类

**继承 `TextSplitter` 创建自定义文本分割器，只需实现 `split_text()` 方法，其他方法（如 `split_documents()`、`create_documents()`）会自动调用它。**

`TextSplitter` 是所有文本分割器的基类，定义了标准接口。自定义时重写核心分割逻辑。

## 最小实现示例

```
from langchain_text_splitters.base import TextSplitter
from typing import List
from langchain_core.documents import Document

class MyCustomSplitter(TextSplitter):
    """自定义分割器 - 按句子首字母分割"""
    
    def __init__(self, **kwargs):
        # 可选：自定义参数
        super().__init__(**kwargs)
    
    def split_text(self, text: str) -> List[str]:
        """核心分割逻辑：按句子分割（以. ! ? 结尾）"""
        sentences = text.split('. ')
        sentences = [s.strip() for s in sentences if s.strip()]
        return sentences

# 使用
splitter = MyCustomSplitter(chunk_size=100, chunk_overlap=10)
chunks = splitter.split_text("Hello world. This is a test. Great!")
print(chunks)
# ['Hello world', 'This is a test', 'Great!']
```

## 完整高级示例（带参数和元数据）

```
class SemanticSentenceSplitter(TextSplitter):
    """语义句子分割器 - 支持长度过滤和元数据"""
    
    def __init__(
        self,
        sentence_endings: str = ".!?",  # 句子结束符
        min_length: int = 10,          # 最小句子长度
        **kwargs
    ):
        self.sentence_endings = sentence_endings
        self.min_length = min_length
        super().__init__(**kwargs)
    
    def split_text(self, text: str) -> List[str]:
        """按语义句子分割，过滤短句"""
        sentences = []
        current = ""
        
        for char in text:
            current += char
            if char in self.sentence_endings:
                if len(current.strip()) >= self.min_length:
                    sentences.append(current.strip())
                current = ""
        
        # 添加剩余内容
        if current.strip():
            sentences.append(current.strip())
        
        return sentences

# 测试
splitter = SemanticSentenceSplitter(min_length=5)
docs = splitter.split_documents([
    Document(page_content="Short. This is good. Very short.")
])
print([d.page_content for d in docs])
# ['This is good.', 'Very short.']
```
